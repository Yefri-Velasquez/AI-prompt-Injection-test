Essential Spring Boot Annotations

March 30, 2026 · 6 min read

Annotations are a core part of how Spring Boot works. They replace XML configuration with simple, readable markers on your classes and methods. Here are the ones you'll use most often.

@SpringBootApplication

The main entry point annotation. It combines three annotations into one:

  • @Configuration — marks the class as a source of bean definitions
  • @EnableAutoConfiguration — enables Spring Boot's auto-configuration
  • @ComponentScan — scans the current package for Spring components
@SpringBootApplication
public class MyApp {
    public static void main(String[] args) {
        SpringApplication.run(MyApp.class, args);
    }
}

@RestController

Marks a class as a REST API controller. It combines @Controller and @ResponseBody, meaning every method returns data directly (not a view name).

@RestController
public class UserController {

    @GetMapping("/users")
    public List<User> getUsers() {
        return userService.findAll();
    }
}

@GetMapping, @PostMapping, @PutMapping, @DeleteMapping

Shortcut annotations for mapping HTTP methods to handler methods:

@GetMapping("/items")        // handles GET requests
@PostMapping("/items")       // handles POST requests
@PutMapping("/items/{id}")   // handles PUT requests
@DeleteMapping("/items/{id}")// handles DELETE requests

@RequestMapping

The general-purpose mapping annotation. The shortcuts above are preferred for clarity, but @RequestMapping is useful for class-level path prefixes:

@RestController
@RequestMapping("/api/v1")
public class ApiController {

    @GetMapping("/status")  // maps to /api/v1/status
    public String status() {
        return "OK";
    }
}

@Autowired

Injects a dependency automatically. Spring finds a matching bean and provides it:

@Service
public class OrderService {

    @Autowired
    private PaymentService paymentService;
}

Tip: Constructor injection is preferred over field injection for testability:

@Service
public class OrderService {

    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}

@Service, @Repository, @Component

These mark classes as Spring-managed beans. They all do the same thing technically, but convey intent:

  • @Component — generic Spring bean
  • @Service — business logic layer
  • @Repository — data access layer (also translates database exceptions)

@Value

Injects values from application.properties:

@Value("${app.name}")
private String appName;

@Value("${server.port:8080}")  // with default
private int port;

@PathVariable and @RequestParam

Extract values from the URL:

// /users/42 → id = 42
@GetMapping("/users/{id}")
public User getUser(@PathVariable Long id) { ... }

// /search?q=spring → query = "spring"
@GetMapping("/search")
public List<Result> search(@RequestParam String q) { ... }

@Entity and @Table

JPA annotations for mapping classes to database tables:

@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private String email;
}

Summary

These annotations handle 90% of what you'll need in a typical Spring Boot application. The key idea is that annotations replace configuration files — they tell Spring what your classes do and how they connect, so you can focus on business logic.
