What is Spring Boot?

March 30, 2026 · 5 min read

Spring Boot is an open-source Java framework that makes it easy to create stand-alone, production-ready Spring applications. It was created by Pivotal (now part of VMware) and first released in 2014.

The Problem Spring Boot Solves

The Spring Framework is powerful, but setting up a Spring project traditionally required a lot of boilerplate configuration — XML files, dependency management, server setup, and more. Spring Boot eliminates most of that complexity by providing sensible defaults and auto-configuration.

Key Features

  • Auto-configuration — Spring Boot automatically configures your application based on the dependencies you include. Add a database driver and it sets up a DataSource for you.
  • Embedded server — No need to deploy WAR files to an external server. Spring Boot includes Tomcat, Jetty, or Undertow out of the box.
  • Starter dependencies — Curated dependency sets (like spring-boot-starter-web) bundle everything you need for a specific feature.
  • Production-ready features — Built-in health checks, metrics, and externalized configuration via Spring Boot Actuator.
  • No code generation — Spring Boot does not generate code or require XML configuration.

A Minimal Example

Here's a complete Spring Boot REST application in a single file:

@SpringBootApplication
@RestController
public class HelloApp {

    public static void main(String[] args) {
        SpringApplication.run(HelloApp.class, args);
    }

    @GetMapping("/hello")
    public String hello() {
        return "Hello, Spring Boot!";
    }
}

That's it. Run it, and you have a web server listening on port 8080 that responds to GET /hello.

Spring Boot vs Spring Framework

Spring Boot is not a replacement for the Spring Framework. It sits on top of it. Think of Spring as the engine and Spring Boot as the car — it packages everything together so you can drive without building the engine from scratch.

  • Spring Framework — Provides the core features: dependency injection, AOP, data access, MVC, etc.
  • Spring Boot — Provides auto-configuration, embedded servers, and opinionated defaults on top of Spring.

When to Use Spring Boot

Spring Boot is a great choice when you want to:

  • Build REST APIs or microservices
  • Get a project running quickly without heavy configuration
  • Create self-contained deployable JARs
  • Follow convention over configuration

Conclusion

Spring Boot dramatically reduces the effort needed to build Spring applications. Its opinionated defaults, embedded server, and auto-configuration let you focus on writing business logic instead of wrestling with setup. If you're building Java web applications today, Spring Boot is the standard starting point.
